📋 Visão Geral

Este arquivo Docker Compose configura dois serviços do Google Tag Manager (GTM) Server-side: um ambiente de produção e outro de preview, ambos integrados com Traefik como proxy reverso para SSL/TLS automático.

💡 Dica: Esta configuração permite testar mudanças no ambiente de preview antes de aplicá-las em produção, garantindo maior segurança e controle.

📦 Estrutura dos Serviços

📦

Serviço de Produção (gtm-server)

URL: https://gtm.emergenciaomni.com.br

Finalidade: Servidor principal para processamento de tags em produção

Modo: Servidor padrão do GTM

🔍

Serviço de Preview (gtm-preview)

URL: https://gtm-preview.emergenciaomni.com.br

Finalidade: Servidor para testes e preview de configurações

Modo: Servidor de preview (RUN_AS_PREVIEW_SERVER=true)

🔀 Integração com Traefik

🌐 Configuração de Rede

YAML 
networks:
  traefik_net:
    external: true
    name: network_public
  • Nome Lógico: traefik_net (usado internamente no compose)
  • Nome Real: network_public (rede Docker externa do Traefik)
  • Tipo: Externa (criada previamente)

🏷️ Labels de Configuração

Serviço de Produção

YAML Labels 
labels:
  - "traefik.enable=true"
  - "traefik.docker.network=network_public"
  - "traefik.http.services.gtm-server-prod.loadbalancer.server.port=8080"
  - "traefik.http.routers.gtm-server-prod.rule=Host(\`gtm.emergenciaomni.com.br\`)"
  - "traefik.http.routers.gtm-server-prod.service=gtm-server-prod"
  - "traefik.http.routers.gtm-server-prod.entrypoints=websecure"
  - "traefik.http.routers.gtm-server-prod.tls.certresolver=letsencryptresolver"

🔄 Fluxo de Funcionamento

1. Inicialização

  1. Docker Swarm cria os containers usando a imagem GTM
  2. Containers se conectam à rede network_public
  3. Traefik detecta os containers através das labels
  4. Certificados SSL são gerados automaticamente via Let's Encrypt

2. Roteamento de Tráfego

Cliente
→
Traefik (443)
→
Container GTM (8080)
Roteamento 
# Produção
https://gtm.emergenciaomni.com.br → gtm-server:8080

# Preview
https://gtm-preview.emergenciaomni.com.br → gtm-preview:8080

📄 Docker Compose Completo

Aqui está o arquivo docker-compose.yml completo para o setup do GTM Server-side:

docker-compose.yml 
version: '3.8'

services:
  gtm-server:
    image: gcr.io/cloud-tagging-10302018/gtm-cloud-image:stable
    environment:
      CONTAINER_CONFIG: aWQ9R1RNLVBWSllRUUYmZ2NzPWd0YWctbWFuYWdlciZndD1ndG0tTnEyc0dQJmE9MzU0MzgxMjQwJmM9d2ViJmQ9MQ%3D%3D
      PORT: 8080
      TZ: America/Fortaleza
      PREVIEW_SERVER_URL: https://gtm-preview.emergenciaomni.com.br
    deploy:
      replicas: 1
      restart_policy:
        condition: on-failure
    networks:
      - traefik_net
    labels:
      - "traefik.enable=true"
      - "traefik.docker.network=network_public"
      - "traefik.http.services.gtm-server-prod.loadbalancer.server.port=8080"
      - "traefik.http.routers.gtm-server-prod.rule=Host(\`gtm.emergenciaomni.com.br\`)"
      - "traefik.http.routers.gtm-server-prod.service=gtm-server-prod"
      - "traefik.http.routers.gtm-server-prod.entrypoints=websecure"
      - "traefik.http.routers.gtm-server-prod.tls.certresolver=letsencryptresolver"

  gtm-preview:
    image: gcr.io/cloud-tagging-10302018/gtm-cloud-image:stable
    environment:
      CONTAINER_CONFIG: aWQ9R1RNLVBWSllRUUYmZ2NzPWd0YWctbWFuYWdlciZndD1ndG0tTnEyc0dQJmE9MzU0MzgxMjQwJmM9d2ViJmQ9MQ%3D%3D
      PORT: 8080
      TZ: America/Fortaleza
      RUN_AS_PREVIEW_SERVER: "true"
    deploy:
      replicas: 1
      restart_policy:
        condition: on-failure
    networks:
      - traefik_net
    labels:
      - "traefik.enable=true"
      - "traefik.docker.network=network_public"
      - "traefik.http.services.gtm-preview-svc.loadbalancer.server.port=8080"
      - "traefik.http.routers.gtm-preview-svc.rule=Host(\`gtm-preview.emergenciaomni.com.br\`)"
      - "traefik.http.routers.gtm-preview-svc.service=gtm-preview-svc"
      - "traefik.http.routers.gtm-preview-svc.entrypoints=websecure"
      - "traefik.http.routers.gtm-preview-svc.tls.certresolver=letsencryptresolver"

networks:
  traefik_net:
    external: true
    name: network_public
💡 Dica: Salve este conteúdo em um arquivo chamado docker-compose.yml e execute docker stack deploy -c docker-compose.yml gtm-stack para fazer o deploy.

🔧 Personalização

Para usar este arquivo em seu ambiente, você precisa ajustar:

  • CONTAINER_CONFIG: Substitua pela configuração Base64 do seu container GTM
  • Domínios: Altere gtm.emergenciaomni.com.br e gtm-preview.emergenciaomni.com.br para seus domínios
  • Timezone: Ajuste TZ conforme sua localização
  • Rede: Verifique se a rede network_public existe em seu Traefik
  • CertResolver: Confirme o nome do seu resolver Let's Encrypt

📝 Exemplo de Uso

Bash 
# 1. Criar o arquivo
nano docker-compose.yml

# 2. Colar o conteúdo e ajustar as configurações

# 3. Fazer o deploy
docker stack deploy -c docker-compose.yml gtm-stack

# 4. Verificar se os serviços subiram
docker service ls

# 5. Verificar logs se necessário
docker service logs gtm-stack_gtm-server

⚠️ Problemas Comuns

Container não inicia
Verificar: CONTAINER_CONFIG válido
Solução: Recriar configuração no GTM e recodificar em Base64
Erro 502 Bad Gateway
Verificar: Container respondendo na porta 8080
Solução: Verificar logs do container e configuração Traefik
Certificado SSL não funciona
Verificar: Propagação DNS
Solução: Aguardar DNS e verificar logs do Traefik
